# Contributing to EONET - Natural Event Tracker

Thanks for your interest in contributing! This guide will help you get started.

## Prerequisites

- **Node.js** 22+
- **Bun** 1.3.9+

## Local Setup

1. Fork and clone the repository:

```bash
git clone https://github.com/reubenwilliam/natural-event-tracker
cd natural-event-tracker
```

2. Install dependencies:

```bash
bun install
```

3. Start the dev server

```bash
bun dev
```

## Code Style

- **Linter**: eslint (run `bun lint`)
- Use decriptive names for files.
- Follow existing code patterns
- Test the production build before committing your changes

## Pull Request Workflow

1. Fork the repo and create a branch from `main`.
2. Make your changes and commit
3. Push to your branch and open a PR against `main`
4. Fill out a clear description of what changed and why

## Commit Convention

Use clear, descriptive commit messages:

- `feat:` - new feature
- `fix:` - bug fix
- `refactor:` - code refactoring
- `chore:` - tooling changes
